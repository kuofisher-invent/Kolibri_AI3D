# 《Kolibri Ai3D 鋼構模式工具列與右側屬性面板規劃》v1.0

---

## 🎯 目標

在現有 Kolibri Ai3D UI 架構下，新增「鋼構模式（Steel Mode）」：

- 不破壞原有 SketchUp 式操作體感
- 提供 Tekla 等級的構件語意建模能力
- 支援 AI Inference 2.0（推論輔助建模）

---

## 🧠 核心設計原則

### 1️⃣ 模式分離（最重要）

- 建模模式（Modeling Mode）
- 鋼構模式（Steel Mode）

👉 不混在同一工具列

---

### 2️⃣ 語意優先（不是幾何）

鋼構模式下：

- 不再畫線 → 建立構件
- 不再推拉 → 指定斷面

---

### 3️⃣ UI 輕量但專業

- 左側：工具（動作）
- 右側：屬性（工程參數）

---

## 🧩 UI 架構

```
[模式切換]
建模 | 鋼構

左側工具列（依模式切換）

中間：3D 視窗

右側：屬性面板（依選取物件切換）
```

---

## 🧭 模式切換設計

### 位置建議

放置於左側工具列最上方：

```
┌──────────┐
│ 建模 | 鋼構 │
└──────────┘
```

---

## 🧰 鋼構工具列設計（左側）

### 第一版（MVP）

| 工具 | 功能 |
|------|------|
| Grid | 建立軸線 |
| Column | 建立柱 |
| Beam | 建立梁 |
| Brace | 建立斜撐 |
| Plate | 建立鋼板 |
| Connection | 建立接頭 |

---

### 工具操作邏輯

#### Column

1. 點擊基準點
2. 輸入或拖曳高度
3. 選擇 Profile

---

#### Beam

1. 點起點
2. 點終點
3. 自動生成構件

---

#### Plate

1. 畫矩形
2. 設定厚度

---

#### Connection

1. 選兩個構件
2. 套用接頭規則

---

## 🧱 工具列圖示設計建議

| 工具 | 圖示 |
|------|------|
| Beam | H 型鋼橫向 |
| Column | H 型鋼直向 |
| Brace | 斜線 |
| Plate | 矩形板 |
| Grid | 十字軸 |
| Connection | 節點 + 螺栓 |

## 🤖 AI Inference 整合

### 在鋼構模式下提供：

- 自動對齊軸線
- 建議梁方向
- 自動偵測柱位
- 建議標準高度

## 🔄 使用流程

```
1. 切換到鋼構模式
2. 建立 Grid
3. 放置 Column
4. 建立 Beam
5. 套用 Connection
6. 調整屬性
```

---

## ⚠️ 常見錯誤避免

| 錯誤 | 解法 |
|------|------|
| 工具太多 | 控制在 6~8 個 |
| UI 過重 | 屬性面板動態顯示 |
| 幾何 vs 語意混亂 | 分模式 |
